## Codebase Patterns
- Project uses `src/` directory: components in `src/shared/components/`, app in `src/app/`
- shadcn UI components go in `src/shared/components/ui/`
- Section components go in `src/shared/components/sections/`
- Layout components go in `src/shared/components/layout/`
- Creative/decorative components go in `src/shared/components/creative/`
- Data files go in `src/data/`
- Blog content goes in `content/blog/` (project root)
- `mdx-components.tsx` lives in `src/` (inside src since project uses src dir)
- next.config.ts uses `createMDX` wrapper from `@next/mdx`
- Motion (not framer-motion) is the animation library — import from `"motion/react"`
- Typecheck: `npx tsc --noEmit`
- Lint: `npm run lint` (biome)
- page.tsx and layout.tsx must NEVER be "use client" (Server Components only)
- shadcn base-nova style uses `@base-ui/react` primitives (NOT Radix UI)
- Biome lineWidth is 220, indentStyle is tab, trailingCommas "none", semicolons "asNeeded"
- Biome enforces a11y rules strictly — avoid `role="region"` on div (use `<section>`), avoid `role="separator"` on div (use `<hr>`)
- Hooks go in `src/shared/hooks/`
- embla-carousel-react is installed for carousel component
- Tailwind v4 uses `@utility` directive to register custom utilities (not `@layer utilities`)
- Creative components that use only CSS animations don't need "use client"
- For Motion-based reduced motion support, use `useReducedMotion()` hook from "motion/react"
- Biome auto-fix: `npx biome check --write <file>` to fix formatting
- For `useEffect` depending on pathname: reference `void pathname` inside the effect body to satisfy biome's exhaustive-deps rule
- Sheet (Drawer) component has nested Root issue — for custom drawers, use Motion + manual state instead
- Biome forbids array index keys — when duplicating arrays for seamless loops, use two separate `.map()` calls with unique key prefixes
- `buttonVariants` lives in `button-variants.ts` (no "use client") — import from there in server components, from `button.tsx` in client components
- Fonts use `next/font/local` with woff2 in `public/fonts/` (not Google Fonts) for offline builds
- Project data: `src/data/projects.ts` exports `projects` array, `Project` type, and utility functions (`getProjectBySlug`, `getFeaturedProjects`, `getProjectsByCategory`, `getAllCategories`)
- CursorSpotlight has pointer-events conflict — always pass `className="pointer-events-auto"` when using it
- To make a section span full viewport behind the fixed navbar, use `-mt-16 pt-16`
- Biome rejects `<a>` in base-ui `render` prop — use `buttonVariants()` with plain `<a>` or `<Link>` instead of `<Button render={<a />}>`
- Next.js 16 dynamic routes: `params` is a Promise — use `await params` in page components and `generateMetadata`
- JSON-LD structured data uses `dangerouslySetInnerHTML` — add `biome-ignore lint/security/noDangerouslySetInnerHtml` comment
- Root layout `metadataBase` enables relative OG image URLs across all pages
- `robots.ts` and `sitemap.ts` in `src/app/` generate `/robots.txt` and `/sitemap.xml`
