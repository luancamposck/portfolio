## Codebase Patterns
- Use `next/font/local` instead of `next/font/google` ŌĆö Google Fonts cannot be fetched at build time in this environment. Font files go in `public/fonts/`.
- Font CSS variable chain: layout.tsx sets `--font-varela-round` on `<html>` via `varelaRound.variable` class ŌåÆ globals.css `@theme inline` maps `--font-sans: var(--font-varela-round)` ŌåÆ `html` applies `@apply font-sans`
- Geist font files already exist in `public/fonts/` (geist-latin.woff2, geist-mono-latin.woff2)
- This codebase uses tabs for indentation ŌĆö when using Edit tool, match with unique string fragments rather than leading whitespace
- Biome is the linter/formatter ŌĆö run `npx biome check --fix src/` after edits
- Responsive mobile-first pattern: `gap-3 sm:gap-6`, `p-4 sm:p-6`, `text-2xl sm:text-3xl` etc.
- `buttonVariants()` outputs static classes ŌĆö cannot be made responsive; use default size or custom responsive className
- Next.js 16 renamed `middleware.ts` to `proxy.ts` ŌĆö export function `proxy` (not `middleware`), file goes at `src/proxy.ts`
- next-intl v4 setup: `src/i18n/routing.ts` (defineRouting), `src/i18n/request.ts` (getRequestConfig), `src/i18n/navigation.ts` (createNavigation)
- next-intl plugin chain in next.config.ts: `withMDX(withNextIntl(nextConfig))` ŌĆö withNextIntl wraps inner, withMDX wraps outer
- Translation files in `messages/{locale}.json` ŌĆö loaded dynamically in `src/i18n/request.ts`
- Root layout (`src/app/layout.tsx`) handles fonts + html/body only; `[locale]/layout.tsx` handles ThemeProvider, Navbar, Footer, NextIntlClientProvider
- `setRequestLocale(locale)` must be called in `[locale]/layout.tsx` for static rendering to work
- `params` is always `Promise<{...}>` in Next.js 16 ŌĆö always `await params` in pages and layouts
