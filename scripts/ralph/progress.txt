## Codebase Patterns
- Use `next/font/local` instead of `next/font/google` â€” Google Fonts cannot be fetched at build time in this environment. Font files go in `public/fonts/`.
- Font CSS variable chain: layout.tsx sets `--font-varela-round` on `<html>` via `varelaRound.variable` class â†’ globals.css `@theme inline` maps `--font-sans: var(--font-varela-round)` â†’ `html` applies `@apply font-sans`
- Geist font files already exist in `public/fonts/` (geist-latin.woff2, geist-mono-latin.woff2)
- This codebase uses tabs for indentation â€” when using Edit tool, match with unique string fragments rather than leading whitespace
- Biome is the linter/formatter â€” run `npx biome check --fix src/` after edits
- Responsive mobile-first pattern: `gap-3 sm:gap-6`, `p-4 sm:p-6`, `text-2xl sm:text-3xl` etc.
- `buttonVariants()` outputs static classes â€” cannot be made responsive; use default size or custom responsive className
- Next.js 16 renamed `middleware.ts` to `proxy.ts` â€” export function `proxy` (not `middleware`), file goes at `src/proxy.ts`
- next-intl v4 setup: `src/i18n/routing.ts` (defineRouting), `src/i18n/request.ts` (getRequestConfig), `src/i18n/navigation.ts` (createNavigation)
- next-intl plugin chain in next.config.ts: `withMDX(withNextIntl(nextConfig))` â€” withNextIntl wraps inner, withMDX wraps outer
- Translation files in `messages/{locale}.json` â€” loaded dynamically in `src/i18n/request.ts`
- Root layout (`src/app/layout.tsx`) handles fonts + html/body only; `[locale]/layout.tsx` handles ThemeProvider, Navbar, Footer, NextIntlClientProvider
- `setRequestLocale(locale)` must be called in `[locale]/layout.tsx` for static rendering to work
- `params` is always `Promise<{...}>` in Next.js 16 â€” always `await params` in pages and layouts
- Translation namespaces mirror component names: `useTranslations('hero')`, `useTranslations('navbar')`, `getTranslations('projectsPage')` etc.
- ICU message format for dynamic values: `t('copyright', { year })`, `t('imageAlt', { title, index })`
- All internal links use `Link` from `@/i18n/navigation` (not `next/link`) â€” auto-prepends locale
- `usePathname()` from `@/i18n/navigation` returns path without locale prefix â€” active state comparisons use clean paths like `/projects`
- Server components needing translations must be `async` and use `getTranslations()` from `next-intl/server`
- Pages with translated metadata use `generateMetadata()` (not static `export const metadata`) + `setRequestLocale(locale)` inside
